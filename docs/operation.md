# 運営計画（勉強会の進め方）

## 目的

受講者は「ゼロから作る Deep Learning」で理論は履修済み．足りないのは **自分で PyTorch のコードを書いた経験**．
本勉強会は，知っている理論を **実際に手を動かして動くコードに落とし込む経験**を積む場とする．
理論の再説明や，フレームワーク内部の細かい仕組みの深掘りはしない（既知のため）．代わりに次を重視する：

- PyTorch の書き方を，自分の手で書けるようになる（`nn.Module` / `DataLoader` / 学習ループ など）
- ゼロから作るでは扱わない **実務スキル**（実験管理 wandb，checkpoint，再現性，計算資源の使い方，エラーの読み方）

## 全体構成

1 モジュール = 1 回．ボトムアップで「動くパイプライン」を積み上げる．
モジュール順: **data → model → 学習ループ → 評価 → 性能向上**．

- **第 1 回（キックオフ）**: 全体の構成と PyTorch の書き方の概要を説明し，「全体でこれを作る」という像を共有．
  ノートブックを一緒に動かして全体の流れを掴む．最後に最初のモジュール（data）に着手し，宿題につなぐ．
- **第 2 回以降（イテレーション）**: 以下のサイクルを各モジュールで繰り返す．
  1. **宿題（前回の終わりに出す）**: その回のモジュールを Python で実装してくる（ノートブックの内容を `src/kws/` に整理）
  2. **回の頭（発表）**: 前回学んだこと・書いたコードを各自が短く共有
  3. **回の後半**: 次のモジュールへ進む．少し座学 ＋ ノートブックでハンズオン → 次の宿題へ
- **最終回**: パフォーマンスチューニング大会（自走）．

回数は **全 6 回程度**（キックオフ 1 ＋ 各モジュール 4 ＋ チューニング大会 1）を目安に，進度で前後させる．

## 1 回のテンプレート（90 分目安）

| フェーズ | 時間 | 内容 |
|---|---|---|
| 発表 | 〜15分 | 前回の宿題で書いたコード・詰まった所・使って分かったコツを共有（人数次第でローテ） |
| 少し座学 | 〜15分 | 今日のモジュールが **何を受け取って何を返すか**，PyTorch でどう書くかの見当をつける |
| ハンズオン | 〜50分 | ノートブックの穴埋めを埋めて動かす．**対面で巡回し，詰まりをその場で拾う** |
| まとめ | 〜10分 | 今日の到達確認 ＋ 宿題（`src/kws/` への整理）の指示 |

要点: 一番むずかしく価値がある「書く」作業を，**対面（巡回サポートあり）で書き始め**，宿題で仕上げる．

## 発表（前回分の共有）

- 内容: **自分が実装したコード／詰まった所とどう解決したか／PyTorch のコツを 1 つ**
- 形式: スライド 3 枚程度・手書き可・5 分．理論のまとめにはしない（既知のため）
- ねらい: 実装経験の共有 ＋ 発表の練習．正確さは指導者がその場でフォローする

## 演習に必ず入れる

### 1. パラメータ変更ハンズオン
`lr` / `batch_size` / `epochs` / `n_mels` などを変えて，**学習曲線（`exp/<run>/loss_curve.png`・wandb）がどう変わるか**を見る．
「変える前に予想 → 実際に回して確認」の順にすると気づきが大きい．

### 2. 実務 Tips（独立した回にせず，関係する場面に散りばめる）
- **学習を回す回（学習ループ）**: `perf-tracker` で空いている GPU マシンを探す → `tmux` でジョブを投げて回しっぱなしにする → `nvidia-smi` で GPU 使用率・VRAM・誰が使っているかを見る
- **データのダウンロードや前処理の待ち時間**: `htop` で CPU・メモリの様子を見る
- **評価・再現の回**: seed 固定，`uv.lock`，run ディレクトリ規約

## 既存教材の使い方

- **ノートブック**（`notebooks/01〜05`）: 授業中に穴埋めして動かす．`main` が穴埋め版，`ans` が完成・実行済み版
- **`src/kws/`**: 宿題でノートブックの内容を整理して実装する．詰まったら解答を取り出せる:
  `git checkout ans -- src/kws/<file>`（写す前に「何をしているか」を声に出すルールにすると写経で終わらない）
- **baseline**: 完成形の参照（test accuracy ≈ 0.92）．最終回のチューニング大会の基準にする
- **各回の内容の詳細**: `docs/curriculum.md`

## 運営の注意

- **環境構築（`uv sync`・データダウンロード・wandb login）は別枠の事前準備**にして，本番の時間に持ち込まない
- 対面はとにかく **巡回**して詰まりを拾う（5〜6 名ならこれが一番効くレベル差対策）
- **速い人にはおまけ課題**（例: ConvBlock を 1 段増やすと shape・パラメータ数はどう変わるか）を用意して手持ち無沙汰を防ぐ
- 宿題（実装）が進まない人が出ると次回の発表が崩れるので，**ハンズオンで「動く所まで」を必ず到達**させてから家に送り出す
