# セッション詳細の作り方ガイド（`authoring_guide.md`）

> 各回の詳細ドキュメント（`session*_*.md`）を作成・改訂するときの標準プロセス．
> session0〜session1 の作成過程で確立した方法をまとめたもの．

## 1. 入力を揃える（下読み）

新しい回の詳細を書き始める前に，以下を全部読む：

- **`docs/sessions/index.md`**：全体構成（各回の位置づけ・前提・成果物）
- **`docs/course_design.md`**：設計思想（解答非公開・実演方式・学習目標）
- **`docs/handoff.md`**：引き継ぎ事項
- **`docs/operation.md`**：運営・当日オペの方針
- **前回のセッション詳細**（特に「宿題」節と「資料作成者向けメモ」＝次回の入口条件）
- **ソースコード**：その回で触る `src/kws/*.py` のスケルトン・TODO・確認スクリプトを確認し，受講者が実際に何を埋めるのかを把握

## 2. 叩き台を書く

以下の**共通構造**で草稿を作成する：

1. 冒頭（ねらい・関連リンク）
2. タイムボックス表（時間配分の概算）
3. ハンズオン①②③…（番号付きの活動ブロック）
4. 確認スクリプト（check_*）の内容と PASS の見え方
5. 宿題（次回への引き継ぎ）
6. 速い人向け（オプション課題）
7. 資料作成者向けメモ（受講者には出さない）

判断に迷うところには `> 【要確認】` や `> tenk コメント：` を残し，決定を先送りにしておく．

## 3. tenk レビュー（方針の確定）

- tenk が草稿を読み，**教育的判断・構成の大枠**を決定する
- tenk のコメントは**最優先**（ペルソナレビューより上位）
- 典型的な判断：どの活動をどのセクションに置くか，何を当日やり何を宿題に送るか，時間配分の調整

## 4. 6ペルソナレビュー（多角的な穴埋め）

Sonnet エージェント 6 体を**並列で**走らせ，以下の視点から草稿をレビューする：

| ペルソナ | 視点 | 重点 |
|---|---|---|
| **受講者** | 認知負荷・詰まりポイント | 「何が分からないか分からない」の発見 |
| **指導者** | 当日オペの実現性 | 時間配分・フォールバック・TA の動き方 |
| **ML専門家** | 技術的正確性 | shape 計算・dtype・再現性・数式の正しさ |
| **教育設計** | 学習目標と活動の整合 | 足場かけの過不足・ブルームのタキソノミー |
| **批判者** | 矛盾・曖昧さ | 「解答非公開」方針との整合・前後の回との矛盾 |
| **再現性** | 受講者間で結果が揃うか | 定数の固定・乱数シード・leaderboard 前提 |

各ペルソナの指摘は **CRITICAL / SHOULD / NICE** の3段階で分類する．

> 回によってペルソナの重みは変わる（モデル回なら ML 専門家の比重が上がるなど）が，6体全部走らせる基本構造は同じ．

## 5. 統合（tenk 方針 + ペルソナ指摘を合成）

- **tenk 方針を骨格**にして，ペルソナの指摘を肉付けする
- 統合時の判断例：
  - tenk「この活動を外す」＋ 教育設計「宿題に送る」→ 宿題に追加
  - 再現性「前処理を固定せよ」→ 具体的な実装仕様（切り詰め位置など）を確定
  - 指導者「Jupyter でハングする」＋ 受講者「エラーで詰まる」→ エラー早見表に追加
- **技術的な裏取り**をこの段階で行う（shape 計算の実測確認・API の挙動確認など）

## 6. 反映・コミット

1. 計画をプランファイルに書き出し，tenk に確認を取る
2. 編集を実行
3. 検証：`>` 仮コメントが消えているか・旧記述が残っていないか grep で確認
4. `index.md` との整合を突き合わせ
5. コミット（Co-Authored-By は付けない）

## 確立した設計原則（全回共通）

| 原則 | 説明 | 例 |
|---|---|---|
| **答え＝抽象例** | 受講者が書く実装は骨組み＋`...`で配布．模範解答は配布しない | `pad_or_trim`, `collate_fn`, `ConvBlock` |
| **探索＝動く例** | API の使い方や探索的な内容は動くコードを見せる | `SPEECHCOMMANDS()`, `MelSpectrogram` |
| **当日は notebook 完結** | `.py` への移植は宿題に送る（editable 設定込み） | session1 で確定 |
| **定数は変えない** | leaderboard・後の回の前提を崩さない | `n_mels=64`, `LABELS` 順序, `n_fft=400` |
| **確認スクリプトの模範解答は非公開** | check_* は PASS 条件のみ配布 | `check_data`, `check_model` |
| **`num_workers=0`（notebook）** | Jupyter でのマルチプロセスハング回避 | DataLoader 呼び出し時に明示 |
